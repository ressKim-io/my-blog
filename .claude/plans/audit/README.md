# 블로그 감사(audit) 작업 — 세션 복구 가이드

> 이 폴더는 블로그 196편 전수 감사 작업의 상태·중간결과를 저장합니다
> 세션이 중단되면 이 문서만 읽고 `state.md`로 이동해서 이어서 진행합니다

## 작업 목표

사용자 요구 요약 (2026-04-23 kickoff)

1. **품질 평균 올리기** — 중복/저가치 글 정리 (병합/삭제/리라이트/유지)
2. **의사결정 서사 일관성 확보** — 단순 트러블슈팅 제외한 글은 `문제상황 → 대안 ABCD → 시도 → 실패사유/선택이유(+프로젝트 맥락)` 구조 갖추기
3. **"우리 프로젝트에 맞게"의 why 강화** — 외부 현실(예: AWS 리전 장애, 대전 화재) → 우리 요구(대용량/무중단) → 그래서 GCP·Cloudflare 도입, 같은 맥락 일관성
4. **날짜 재분배** — 3~4월에 몰린 글 중 일부를 2월로 이동 (실제 작성일과 괴리 허용)

## 파일 구조

- `README.md` (이 문서) — 프로젝트 개요, 복구 시작점
- `state.md` — **현재 진행상태, 마지막 세션 요약, 다음 액션** (세션 시작 시 필독)
- `plan.md` — 전체 Phase 계획
- `criteria.md` — 판정 기준 (중복/저가치/서사구조/컨텍스트why)
- `inventory.md` — 196편 전수 메타데이터 (slug, title, category, date, series, length 등)
- `findings-<category>.md` — 카테고리별 진단 결과 (Phase 1)
- `decisions.md` — 유지/병합/삭제/리라이트/날짜이동 결정 (Phase 2)

## 세션 복구 절차

1. 이 README를 읽어 목표를 상기한다
2. `state.md`를 읽어 "어디까지 했는지 / 다음 뭐 할지"를 확인한다
3. `plan.md`에서 현재 Phase의 체크리스트를 확인한다
4. 필요한 파일(criteria, inventory 등)을 읽고 작업을 이어간다
5. 작업 후 반드시 `state.md`를 업데이트한다

## 작업 원칙

- 이 세션에서는 **진단 문서만 생성** (src/content 파일은 건드리지 않음)
- Phase 2(결정)·Phase 3(실행)는 사용자 승인 후 진행
- 모든 중간 결과는 파일로 저장해 세션 간 이어짐 보장
