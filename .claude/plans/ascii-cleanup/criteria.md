이번 세션은 **평탄화 + 보존** 두 갈래만. 시각화 도구(D2/Mermaid/drawio/HTML→PNG/MDX 컴포넌트) 도입은 **다음 세션(디자인 개편 + 다크모드 제거)**에서 일관된 시스템으로 한 번에 결정.

근거:

- Mermaid/drawio/HTML→PNG는 디자인 시스템 결정 전에 도입하면 톤 불일치 + 재작업 위험
- 다크모드 제거 후 다이어그램 톤/색상 자유도 ↑ → 다음 세션에서 결정 효율 ↑

## 자동 분류 → 추천 결정 매트릭스

`scripts/scan.mjs`의 `recommendDecision()`이 다음 룰로 자동 부여:

| kind | 조건 | decision | 이유 |
|---|---|---|---|
| `tree` | `├──`/`└──` 토큰 ≥3 + 박스 없음 | **keep** | 디렉토리/의존성 트리는 ASCII가 자연스러움 |
| `code-arrow` | 코드 lang(go/bash/yaml 등) + 박스 없음 | **skip** | 슈도코드/주석. 다이어그램 의도 아님 |
| `flow-diagram` (>5줄) | 박스 + 화살표 + 6줄 이상 | **keep** | 시각화 가치 있음 → 디자인 개편 시 시스템 변환 |
| `architecture` (>5줄) | 박스 다수 + 6줄 이상 | **keep** | 동상 |
| `flow-diagram`/`architecture` (≤5줄) | 작은 박스 다이어그램 | **flatten** | 표/문장으로 충분 |
| `box-small` | 박스 + 14줄 이하 | **flatten** | 정보량 적음, 표/문장 우월 |
| `arrow-only` | text/no-lang 블록의 화살표 | **flatten** | 보통 짧은 흐름, 문장으로 풀어쓰기 |
| `misc` | 위에 안 잡힘 | **flatten** | 기본값 |

## 평탄화(flatten) 방법론

블록 유형별로 다음 패턴 우선 적용. 자동 변환 없음, 글 컨텍스트 보고 사람이 판단.

### 1. 단순 박스 → 인용/콜아웃 또는 삭제

```
❌ 변환 전:
```
┌─────────────┐
│  Concept X  │
└─────────────┘
```

✅ 변환 후 (개념 강조가 필요하면):
> **Concept X** — 한 줄 설명

✅ 또는 본문 흐름에 녹여서 그냥 삭제
```

### 2. 박스 비교 (Sidecar vs Ambient 류) → markdown 표

```
✅ 변환 후:
| 항목 | Sidecar | Ambient |
|---|---|---|
| ... | ... | ... |
```

### 3. 짧은 화살표 흐름 (`A → B → C`) → 인라인 또는 번호 목록

```
❌ 변환 전:
```
Request → Auth → Backend → Response
```

✅ 변환 후 A (인라인): "요청은 Auth → Backend 순으로 흐릅니다."
✅ 변환 후 B (번호): "1. Auth가 토큰을 검증합니다. 2. Backend가 ..."
```

### 4. 디렉토리 트리 (작은 것) → 코드블록 lang 명시

```
✅ 변환 후:
```text
src/
├── content/
└── components/
```
(language 미지정 코드블록은 CLAUDE.md 위반. `text` 명시)
```

### 5. 정보량 0인 장식성 박스 → 삭제

```
❌ 변환 전:
```
═══════════════════
   PART 1: 시작
═══════════════════
```

✅ 그냥 markdown `## PART 1: 시작` 헤딩으로
```

## 보존(keep) 처리 규칙

- **본문 수정 금지**: 글 본문은 건드리지 않음
- **목록만 유지**: `inventory.md`의 keep 블록 105개가 다음 세션 작업 대상
- **디자인 개편 세션 인풋**: 새 디자인 시스템으로 일관 변환 (MDX 컴포넌트 / drawio / Mermaid 등 그때 결정)
- **신규 키프 추가 금지**: 평탄화 작업 중 "이건 그림이 더 낫다" 판단 시에도 변환은 디자인 개편 세션으로 미룸

## 작업 단위 (시리즈 락)

- **처리 단위 = 시리즈 또는 카테고리**, 결정 단위 = 블록
- 한 시리즈의 일부 글만 처리하지 않음 (시리즈 일관성)
- 시리즈가 없는 단독 글은 카테고리로 묶음
- 한 commit = 한 시리즈/그룹

## 재발 방지 규칙 (Phase 4에서 CLAUDE.md 추가)

신규 글 작성 시:

| 패턴 | 정책 |
|---|---|
| 박스(`┌┐└┘`) + 5줄 이상 | **금지** — 문장/표로 풀거나, 디자인 개편 후 컴포넌트 |
| 박스 + 5줄 이하 | **지양** — 표/콜아웃 우선 |
| 디렉토리 트리(`├ └`) | **허용** — 단, 코드블록 `text` 명시 필수 |
| 짧은 인라인 흐름(`A → B`) | **허용** — 단, 한 줄짜리만 |
| ASCII 디자인 목업 | **허용** — Design Change Rule 적용 |
