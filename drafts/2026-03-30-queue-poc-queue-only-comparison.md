## 5. 서버 부하 추정 (Redis 부하)

### VU당 Redis 연산 추정

| 연산 | 수연 | 준상 | 성전 |
|------|------|------|------|
| **enter** | HASH write + Lua script | HASH write | ZSET zadd + HASH hset + STRING set (3건) |
| **status polling** | Lua script (읽기) | HASH hget | HASH hget + ZSET zrank (2건) |
| **heartbeat** | 없음 | 없음 | STRING setex (1건, 10초마다) |
| **seat_enter** | Lua script (쓰기) | HASH hset | ZSET zrem + STRING set + HASH hincrby (3건) |
| **leave** | Lua script | HASH update | ZSET zrem + STRING del + HASH hincrby (3건) |
| **만료 체크** | Scheduler 30초 | Redis TTL 자동 | Redis Key Expiration 이벤트 |

### Polling 기반 Redis 읽기 부하

| 구현 | polling 간격 | heartbeat 간격 | VU당 Redis ops/sec | 3000 VU Redis ops/sec |
|------|-------------|---------------|-------------------|----------------------|
| 수연 | 2초 | 없음 | 0.5 (status만) | ~1,500 |
| 준상 | 1초 | 없음 | 1.0 (status만) | ~3,000 |
| 성전 | 5초 | 10초 | 0.3 (status+heartbeat) | ~900 |

> 성전: polling 간격 5초 + heartbeat 10초 → **VU당 Redis 부하 가장 낮음**
> 준상: polling 1초 → Redis 읽기 부하 가장 높음

### 총 HTTP 요청 비교 (서버 부하 직결)

| VU | 수연 reqs | 준상 reqs | 성전 reqs |
|----|-----------|-----------|-----------|
| 300 | - | - | 34,894 |
| 1000 | - | - | 106,987 |
| 3000 | 445,086 | 329,262 | 309,449 |

> 3000 VU: 성전 309K < 준상 329K < 수연 445K — 성전이 **서버 요청 수 가장 적음**

### 네트워크 트래픽

| VU | 수연 recv/sent | 준상 recv/sent | 성전 recv/sent |

## 7. Redis 자료구조 + 승격 메커니즘 비교

| 항목 | 수연 (방식3) | 준상 (방식1) | 성전 (방식2) |
|------|-------------|-------------|-------------|
| **대기열 자료구조** | Hash + Lua script | Hash (O(1) 카운터) | **ZSET** (score=순번) + HASH |
| **순번 관리** | Hash 필드 | Hash 카운터 | **ZSET score** (정렬 보장) |
| **승격 트리거** | status 호출 시 동적 계산 | Scheduler 1초 주기 | **이벤트 기반** (leave/expire) |
| **승격 방식** | publishedRank 매번 재계산 | 빈 슬롯만큼 승격 | **분산 락 획득 → allowedNum 갱신** |
| **Heartbeat** | 없음 | 없음 | **있음** (TTL 30초, 10초 갱신) |
| **만료 처리** | Scheduler 30초 주기 | Redis TTL 자동 | **Redis Key Expiration Event** |
| **분산 락** | Lua script (원자적) | 없음 | **있음** (Redisson) |
| **토큰 방식** | JWT (JWE AES-256) | JWT | Custom AES (@RequestParam) |
| **Redis 키 수 (VU당)** | ~2개 | ~2개 | **~5개** (entry, ZSET, heartbeat, active, status) |

## 10. 추가 확인 필요 사항

Grafana에서 확인해야 할 서버 부하 지표 (K6에서는 측정 불가):
- **Redis**: CPU 사용률, 메모리, ops/sec, connected clients, keyspace hits/misses
- **Pod**: CPU/Memory 사용량 (queue pod 1개 기준)
- **JVM**: GC pause, heap usage, thread count
- **Network**: Pod 간 latency (Istio sidecar 포함)
