## 한국어 카피 원칙 (외부 가이드 + 본 세션 도출)

[테크니컬 라이팅 가이드](https://technical-writing.dev/sentence/natural-kor-expression.html) 핵심 + 본 세션에서 추가로 식별한 패턴들.

### 원칙 1. 명사화 → 동사화

영어는 동사를 명사형(`-tion`, `-ing`)으로 바꿔 쓰는 게 자연스럽지만, **한국어는 동사를 그대로 쓰는 게 자연스럽다.**

| ❌ 영어식 명사형 | ✅ 한국어 동사형 |
|---|---|
| API 키를 이용한 사용자 인증 처리가 완료된 후 | API 키로 사용자를 인증한 후 |
| CDN이 이를 Origin 도달 전에 흡수한 효과입니다 | Origin 도달 전에 흡수되는 것을 확인했습니다 |
| 관측이 진짜 사건 4건을 잡았습니다 | 운영 중 4건을 대시보드가 먼저 알려줬습니다 |

### 원칙 2. 한자어 동사 ("수행/실행/진행") 빼기

| ❌ | ✅ |
|---|---|
| 로그 파일을 삭제하는 작업을 수행합니다 | 로그 파일을 삭제합니다 |
| 모니터링을 실행하여 시스템을 측정합니다 | 모니터링으로 시스템을 측정합니다 |

### 원칙 3. 무생물 주어 회피

영어 "monitoring caught X" 같은 무생물 주어 직역은 한국어에서 어색하다.

| ❌ 무생물 주어 직역 | ✅ 도구로 표현 |
|---|---|
| 관측이 사건을 잡았습니다 | 대시보드로 운영 중 4건을 잡았습니다 |
| 시스템이 결정합니다 | 시스템에서 결정합니다 / 시스템 덕에 결정됐습니다 |

### 원칙 4. "진짜/실제" 같은 비교 암시 강조어 회피

"진짜 X"라고 하면 자동으로 "가짜 X" 비교 대상을 암시한다 → 어색.

| ❌ | ✅ 대안 |
|---|---|
| 진짜 사건 4건을 잡았습니다 | 운영 중 마주친 문제 4건 |
| 실제 운영 사례 | 운영 중 발견한 사례 / 실 운영 사례 |
| 진짜 효과가 있었습니다 | 효과가 있었습니다 (그냥 빼라) |

### 원칙 5. 격식·모호 단어 → 구체 단어

"사건"은 한국어에서 격식적이고 모호하다. 시스템 장애 맥락에서는 더 구체적으로:

| ❌ 모호 | ✅ 구체 |
|---|---|
| 사건 | 문제 / 이상 / 장애 / 이슈 |
| 처리 | 발견 / 잡음 / 해결 / 막음 |
| 운영 | (맥락에 따라) 측정 / 관측 / 감시 |

### 원칙 6. 카피 메시지와 본문 일관성

헤드라인이 **본문이 증명하는 것을 요약**해야 한다. 헤드와 본문이 모순되면 독자가 혼란.

> 예: 헤드라인 "각각 따로 골랐습니다" + 본문 "OTel 표준 통합" → 모순.
> 수정: "컴포넌트는 따로, 시스템은 OTel 답게 하나입니다" — 헤드 안에 분리/통합 양 메시지 박음.

### 원칙 7. 도메인 시작 → 기술 결과로 (포트폴리오 한정)

기술 수치만 박는 카피는 비기술자에게 "그래서 뭐?"가 된다. **도메인 맥락 → 설계 의도 → 검증 결과** 흐름이 자연스럽다.

> ❌ "588K 요청 중 390K (66%)가 Edge에서 처리되었습니다 — polling으로 인한 요청 밀집도의 증거이자, CDN이 Origin 도달 전에 흡수한 효과입니다."
>
> ✅ "티켓팅에서 사용자가 서버로 가장 많이 보내는 요청은 **대기열에서 발생하는 polling**입니다. 이를 CDN Edge에서 받아내도록 설계했고, **588K 요청 중 97%가 Origin 도달 전에 흡수**되는 것을 확인했습니다."

## 적용 사례 (5개 슬라이드 + 슬라이드 순서)

### 1. 슬라이드 순서 재배치 (About → Overview → AI 4장 → 본문)

이전: `01 About → 02~05 AI 4장 → 06 Overview → 본문`
변경: `01 About → 02 Overview → 03~06 AI 4장 → 본문`

**이유**: PDF로만 보면 Goti가 무슨 프로젝트인지 모르는 상태에서 AI hero를 만나 "왜 갑자기?". Overview를 먼저 박으면 "Goti가 뭔지" → "그걸 어떻게 만들었는지(AI)" → "구체 모듈" 흐름이 자연스럽다.

파일명까지 prefix 변경 (`02-overview.html`로 이동, 02~05가 03~06로 시프트). `capture.mjs`가 디렉토리 정렬 순서로 PPTX 생성하므로 파일명 변경만으로 PPTX 순서도 자동 갱신.

### 2. AI 4장 — 소속(framing) 변경

| 위치 | 변경 |
|---|---|
| Hero 헤드라인 | `AI를 어디까지 운영할 수 있을까요?` → **`Goti에서 AI를 이렇게 운영했습니다`** |
| Hero eyebrow | `AI · K8s 운영 자동화` → **`GOTI에서의 AI · 운영 방식의 차별화`** |
| AI Cycle eyebrow | `AI · 자기 개선 사이클` → **`GOTI에서의 AI · 자기 개선 사이클`** |
| AI Harness eyebrow | `AI · 산업 framework 매핑` → **`GOTI에서의 AI · 표준보다 먼저`** |
| AI Meta eyebrow | `DEVOPS 운영 · 시스템은 도구로 활용` → **`GOTI에서의 AI · 운영 영역 정의`** |
| index.html 섹션 라벨 | `▸ AI · 차별화` → **`▸ GOTI에서의 AI 활용`** |

**핵심 통찰**: "AI 4장이 어디에 위치해도 어색"한 진짜 원인은 "AI가 별도 메타 토픽으로 보였기 때문". eyebrow에 `GOTI에서의 AI` prefix를 통일하니 위치 고민 자체가 해소됐다.

또 Hero 폰트 크기도 84px → 64px로 조정 (이전 짧은 카피 기준이라 새 카피에 너무 컸음).

### 3. 슬라이드 02 (Overview) — 카드 3 (모니터링)

| 항목 | 이전 | 이후 |
|---|---|---|
| 제목 | 모니터링 직접 구축 | **비즈니스 + 스파이크 관측** |
| 설명 | LGTM을 우리 시스템에 맞춰 직접 구축... 오진을 막는 대시보드 37종·패널 380개 | **야구 티켓팅에 맞춘 대시보드를 직접 만들었습니다. 실시간 대기열 · 분당 판매 · 스파이크 RPS를 한눈에 봅니다.** |
| 메트릭 | 37 / 380 | **4건** (운영 중 먼저 발견한 문제) |

**원칙 적용**: 단순 양적(37/380)→의미 영역(비즈니스/스파이크). 다른 카드(97.7%, 30s)와 단위가 다양해져 시각적 차별성도 확보.

### 4. 슬라이드 13 (CDN 검증)

본문 카피 변경:

> Before: 588K 요청 중 390K (66%)가 Edge에서 처리되었습니다 — 클라이언트 polling으로 인한 요청 밀집도의 증거이자, CDN이 이를 Origin 도달 전에 흡수한 효과입니다.
>
> After: 티켓팅에서 사용자가 서버로 가장 많이 보내는 요청은 **대기열에서 발생하는 polling**입니다. 이를 CDN Edge에서 받아내도록 설계했고, **588K 요청 중 97%가 Origin 도달 전에 흡수**되는 것을 확인했습니다.

**원칙 적용**:
- 도메인 맥락 시작 (티켓팅의 가장 많은 요청 = 대기열 polling)
- 설계 의도 박음 ("받아내도록 설계했고" — 본인의 의도적 결정)
- 수치 정합성 (헤드라인 97.7%와 본문 97% 일치 — 이전 66%는 충돌)

### 5. 슬라이드 18 (LGTM 선정)

| 위치 | 변경 |
|---|---|
| eyebrow | `선정 · 4개 컴포넌트 독립 평가` → **`선정 · 컴포넌트는 따로, 운영은 OTel 답게`** |
| 헤드라인 | `메트릭 · 로그 · 트레이스 · 수집기, 각각 따로 골랐습니다` → **`컴포넌트는 따로, 시스템은 OTel 답게 하나입니다`** |

**핵심**: 헤드라인이 본문(Exemplar 체인 / 쿼리 통일 / OTel 표준 준수)과 모순됐다. 대조 구조 "컴포넌트는 따로 ↔ 시스템은 하나"로 분리/통합 동시에 박음. `OTel 답게`가 본인 설계 철학으로 박힘. 서브헤드는 헤드라인의 풀어쓰기일 뿐 정보 추가 0이라 제거.

### 빌드 시 주의: stale PNG 정리

파일명 변경 (02-ai-hero → 03-ai-hero 등)으로 이전 캡처 PNG가 stale 상태로 남았다. `make-pptx.mjs`가 `out/` 디렉토리를 정렬해 PPTX 만들기 때문에 stale 파일이 섞이면 잘못된 슬라이드가 들어간다.

```bash
rm -f slides/build/out/*.png && cd slides && npm run all
```
