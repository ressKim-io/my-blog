# 5편. 인터넷의 물리적 실체 — ISP, AS, 그리고 "ISP 건물 안의 CDN"

> **시리즈 "패킷은 어떻게 사용자에게 닿는가"의 5편**
> 4편에서 "네트워크 위치가 속도를 가른다"고 했다. 이번엔 그 "위치"의 정체를 밑바닥부터 판다. 인터넷이 실제로 어떻게 생겼는지.

## "인터넷"은 하나의 망이 아니다

먼저 가장 큰 오해부터. 인터넷은 거대한 단일 네트워크가 아니다. **수많은 독립 네트워크가 서로 연결된 것**이다. *network of networks* — 이게 inter-net(사이-망)이라는 단어의 어원이다.

```
"인터넷" = 독립적인 네트워크 여러 개가 서로 연결된 것

  [KT 네트워크] ──연결── [SKB 네트워크] ──연결── [LGU+ 네트워크]
       │                                              │
       └──연결── [구글 네트워크] ──연결── [클라우드 네트워크]─┘
```

## ISP와 AS

**ISP (Internet Service Provider)** = 인터넷 회선을 깔고 파는 회사. 한국이면 KT, SK브로드밴드, LG U+. 매달 내는 인터넷 요금을 받는 곳이 ISP다.

그리고 위 그림에서 **독립적으로 운영되는 네트워크 하나**를 **AS (Autonomous System, 자율 시스템)** 라고 부른다. 고유 번호(ASN)가 붙는다. KT도, 구글도, 클라우드 사업자도 각자 자기 AS가 있다.

```
AS = 각자 독립적으로 운영하는 네트워크 (클라우드 계정/VPC 같은 것)
  - 내부는 자기 맘대로 운영
  - 서로 통신하려면 경계에서 연결(peering)을 맺음
  - AS끼리 경로를 알려주는 프로토콜 = BGP

★ "ISP 경계를 넘는다" = "AS 경계를 넘는다"  (4편의 그 경계)
```

> 🎨 **그래픽 추천**: AS들을 각각 색이 다른 구름(영역)으로 그리고, 경계에서 BGP로 연결되는 모습. "인터넷은 하나가 아니라 여러 영역의 연합"이라는 메시지가 핵심.

## 인터넷의 물리적 실체

ISP는 추상적인 클라우드가 아니라 **물리적으로 존재한다.** 세 가지로 이뤄진다.

```
① 건물    : 데이터센터 / 국사(局舍). 옛 전화국이 지금 인터넷 국사 역할.
② 케이블  : 땅 밑 광케이블, 도시 간 간선, 해저 케이블
③ 장비    : 그 건물 안의 대형 라우터·스위치
```

이 건물들이 계층 구조로 연결돼 있다.

```
              [백본 / 코어망]   ← ISP의 심장. 초대형 라우터, 초고속 간선
               /      |      \
        [서울 IDC][부산 IDC][광주 IDC]   ← 지역 거점 (대형 데이터센터)
           /   \
    [강남 국사][서초 국사] …   ← 동네 단위 국사
        │
    [동네 단자함 / 광케이블]
        │
    [집 공유기] → [집 PC]   ← ★ 너는 여기. 망의 "맨 끝(edge/leaf)"
```

> 🎨 **그래픽 추천**: 이 계층 트리는 그래픽으로 만들면 매우 직관적이다. 위에서 아래로 뻗는 나무 형태. 맨 아래 leaf(집)에 사람 아이콘을 두면 "내가 망의 어디에 있나"가 와닿는다.

**"ISP 건물 안"** 이라고 하는 건 위쪽의 IDC·국사를 말한다. ISP의 라우터가 실제 랙에 꽂혀 돌아가는 물리적 공간이다.

## 핵심: 집 PC vs 임베디드 캐시 — 같은 망, 다른 위치

여기가 이 글의 정점이다. 둘 다 똑같은 ISP 망(케이블)을 쓰는데, **그 망의 어디에 붙어 있느냐**가 완전히 다르다.

### 집 PC가 콘텐츠를 줄 때 (망의 끝 → 끝)

집 PC는 망의 가장 바깥 끝에 매달린 소비자(customer)다. 누가 네 PC에서 뭘 받으려면, 트래픽이 ISP 코어까지 들어왔다가 **다시 바깥 끝(너집)까지 나갔다 돌아와야** 한다.

```
사용자 → 액세스망 → [ ISP 코어 ] → 다시 액세스망 → 너집 PC
                        ↑                          ↑
                   중심까지 왔다가          또 변두리 끝까지 나감
```

### 임베디드 캐시가 콘텐츠를 줄 때 (망의 중심에서 바로)

CDN 회사는 **자기 서버를 ISP 건물(IDC) 안에 직접 갖다 놓는다.** 그래서 사용자 요청이 코어까지만 오면 거기서 바로 응답한다.

```
사용자 → 액세스망 → [ ISP 코어 옆 CDN 박스 ]   ← 여기서 끝!
                        ↑
                   중심에서 바로 나감. 변두리까지 안 나감.
```

### 그 "CDN 박스"의 실체

CDN 회사가 **실제 서버 장비**(랙에 꽂는 박스 + 디스크 잔뜩)를 ISP 데이터센터로 보내고, ISP 직원이 랙에 설치한 뒤 ISP 내부망에 직접 연결한다.

```
[ISP 데이터센터(IDC) 안의 한 랙]
┌──────────────────────────┐
│  ISP 라우터               │  ← ISP 본인 장비
│  ISP 스위치               │
│ ─────────────────────────│
│  📦 CDN 캐시 박스 (A사)    │  ← CDN이 보낸 물리 서버.
│  📦 CDN 캐시 박스 (B사)    │     ISP 내부망에 직결됨.
│  📦 CDN 캐시 박스 (C사)    │
└──────────────────────────┘
        ↑
  새벽 한가할 때 콘텐츠를 미리 채워둠 (pre-fill).
  낮에 사용자가 찾으면 이 박스에서 바로 나감.
```

> 🎨 **그래픽 추천**: 데이터센터 랙 일러스트에 ISP 장비와 CDN 박스가 같은 랙에 꽂혀 있는 모습. "남의 회사 서버가 ISP 건물 안에 있다"는 충격 포인트를 시각적으로 전달.

이게 **embedded cache(임베디드 캐시)** 다. "사용자 근처"를 넘어 아예 **사용자의 ISP 건물 내부**까지 들어간 것이다. 그러면 ISP 경계(AS 경계)를 *0번* 넘고, 디스크 대신 RAM에서 나가고, 회선도 ISP 내부 전용 고속망이라 — 집 PC가 따라올 수가 없다.

## 비유: 물류 창고

```
중앙 창고에서만 배송:   주문 → 멀리 있는 중앙 물류센터 → 너집 (느림)

지역 물류센터 방식:     전국 곳곳 물류센터에 재고를 미리 깔아둠
                       → 가까운 지역센터에서 바로 출고 (빠름)

  ┌─ 인터넷 도로망     = 모두가 쓰는 전국 도로 (똑같이 씀)
  ├─ ISP 건물(IDC)    = 지역 물류센터
  ├─ CDN 임베디드 캐시 = 그 물류센터에 미리 깔아둔 재고
  └─ 집 PC           = 시골 막다른 골목 끝 집에서 물건 보내는 격
```

도로(망)는 똑같이 쓰지만, **재고를 어디에 두느냐**가 속도를 가른다. 이게 핵심이다.

## "전국에 같은 망인데 CDN은 다르다"는 정확히는?

**"다른 망"이 아니라 "같은 망의 다른 위치 + 다른 회선 등급"이다.**

```
같은 것:  케이블·국사 같은 물리 인프라 (다 같이 씀)

다른 것:  그 망의 "어디에" 붙어 있느냐
  ├─ 집 PC : 변두리 끝(leaf). 소비자용 회선(비대칭·best-effort·공유).
  └─ CDN   : 망의 중심부(IDC, IXP). 전용·대칭 회선.
            때로는 ISP 망 내부에까지 직접 들어가 있음(embedded).
```

고속도로 비유로: 같은 고속도로망을 써도 **나는 시골 끝 마을 진입로**에 살고, **CDN은 나들목(IC) 바로 옆 대형 물류창고**에 있는 것이다.

## CLI로 직접 보기 — traceroute로 계층 보기

이 ISP 계층은 추상적인 게 아니라 직접 볼 수 있다.

```bash
# 목적지까지 거치는 라우터(홉)들을 순서대로 + 각 홉 RTT
traceroute google.com        # 리눅스/맥
tracert  google.com          # 윈도우
mtr google.com               # 홉별 손실률까지 실시간

# 특정 IP가 어느 AS(회사)에 속하는지
whois 8.8.8.8 | grep -iE 'origin|netname|orgname'
```

읽는 법:

```
첫 홉    : 192.168.x.x      → 너집 공유기
다음 몇 홉: ISP 액세스망 → ISP 코어   (RTT가 조금씩 늘어남)
그러다…
  - 대형 서비스(구글 등) → 홉 적고 RTT 거의 안 늘며 도착 → ★ 가까운 캐시/PoP
  - 해외 비-CDN 서버 → 중간에 RTT가 확 튐(바다 건넘) → 거리·홉 비용 노출
```

**실험**: 구글(CDN 있음)과 해외 작은 사이트(CDN 없음)를 각각 traceroute 해보라. 구글은 한국 안에서 몇 홉 만에 낮은 RTT로 끝나는데, 이게 임베디드 캐시/가까운 PoP가 일하고 있다는 증거다.

## 정리

- "인터넷"은 단일 망이 아니라 **독립 네트워크(AS) 수천 개의 연합**이다.
- **ISP**는 회선을 파는 회사고, 물리적으로 **건물·케이블·라우터**로 존재한다.
- 망은 **백본 → 지역 IDC → 국사 → 집**의 계층이고, 집 PC는 망의 **맨 끝(leaf)**이다.
- **임베디드 캐시**는 CDN 박스를 ISP 건물 *안*에 직접 넣은 것 → ISP 경계를 0번 넘는다.
- 같은 망, **다른 위치**. 그게 속도를 가른다.
